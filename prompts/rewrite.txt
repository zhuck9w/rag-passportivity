Ты готовишь поисковый запрос для бота базы знаний компании по программам
гражданства и ВНЖ.

Известные страны базы: [[COUNTRIES]]

История диалога (может быть пустой):
[[HISTORY]]

Новый вопрос сотрудника: [[QUESTION]]

Сделай шесть вещей:
1. Переформулируй вопрос так, чтобы он был полным и понятным без истории:
   подставь страну и программу из контекста, если в вопросе их нет.
   ЗАПРЕЩЕНО добавлять в query сущности, которых нет ни в вопросе, ни в
   истории: типы программ (ВНЖ, гражданство, золотая виза), страны,
   документы. Спрашивали про биометрию — query про биометрию, а не про
   «программу ВНЖ».
2. Перечисли ВСЕ страны, о которых спрашивают. Названия из списка известных
   стран возвращай ТОЧНО как в списке. Если вопрос без привязки к стране —
   верни пустой список. Если спрашивают про страну, которой НЕТ в списке —
   всё равно включи её название, как написал сотрудник.
3. Сам поисковый запрос в "query" формулируй по-русски (база знаний в
   основном на русском), даже если вопрос задан по-английски. Английские
   названия программ и аббревиатуры (Golden Visa, D7, NHR) оставляй как есть.
   Конкретные ЧИСЛА и суммы из вопроса в query НЕ переноси — по цифрам
   поиск не работает; передавай их смысл словами («стоимость программы»,
   «размер взноса», «порог инвестиций»). Опечатки исправляй.
   Пример: «Вануату - 130 165 тысяч это вложкния или отдать безвозвратно?» →
   {"query": "стоимость программы Вануату инвестиции возвратные или безвозвратный взнос", ...}
   Для ОБЗОРНЫХ вопросов (см. п. 5) формулируй query предметно, словами,
   которыми написаны карточки программ: «вид на жительство» полностью, а не
   только «ВНЖ», плюс конкретика — «условия», «инвестиции», «сроки»,
   «требования». Общие слова вида «программы в ЕС» ищутся плохо.
   Для ТОЧЕЧНЫХ вопросов (survey=false) это предметное расширение НЕ
   применяй: только смысл последней реплики плюс недостающий контекст.
4. Определи тематику вопроса — ровно одна из: [[TOPICS]].
   Если не подходит ничего — "другое".
5. Поставь "survey": true|false — является ли вопрос ОБЗОРНЫМ: спрашивает
   про многие страны/программы СРАЗУ, без привязки к одной-двум конкретным.
   survey=true — только для вопросов ПО ТЕМЕ гражданства/ВНЖ/паспортов вида
   «какие вообще есть…», «что доступно…», «в каких странах…», «сравни все…».
   survey=false — для точечных вопросов про конкретную страну или программу
   И ОБЯЗАТЕЛЬНО для любых вопросов НЕ по теме гражданства и ВНЖ (быт, кухня,
   внутренние дела компании, отпуска, погода и прочий оффтоп).
6. Определи "intent" — намерение сообщения, ровно одно из:
   - "meta" — вопрос ПРО САМОГО БОТА: «что ты умеешь», «как тобой
     пользоваться», «какие страны ты знаешь», «кто ты»;
   - "smalltalk" — ЧИСТОЕ приветствие/благодарность/прощание БЕЗ
     содержательного вопроса;
   - "knowledge" — ВСЁ остальное.
   КРИТИЧНО: если в сообщении есть ЛЮБОЙ содержательный вопрос (программы,
   страны, документы, сроки, налоги и т.п.), intent="knowledge", даже если
   сообщение начинается с приветствия или благодарности. Оффтоп-вопросы
   (борщ, погода, отпуска) — тоже "knowledge": это содержательный вопрос,
   пусть база на него и не ответит. При любом сомнении — "knowledge".

Примеры:
«а в Португалии?» после обсуждения порога инвестиций на Мальте →
{"query": "минимальный порог инвестиций по программе Португалии", "countries": ["Португалия"], "topic": "инвестиции и стоимость", "survey": false, "intent": "knowledge"}
«где дешевле ВНЖ — Мальта или Португалия?» →
{"query": "стоимость и минимальный порог инвестиций ВНЖ Мальта Португалия", "countries": ["Мальта", "Португалия"], "topic": "инвестиции и стоимость", "survey": false, "intent": "knowledge"}
«какие есть программы с ВНЖ за инвестиции?» →
{"query": "программа вид на жительство ВНЖ за инвестиции условия и порог", "countries": [], "topic": "другое", "survey": true, "intent": "knowledge"}
«какие вообще опции ВНЖ есть в ЕС сейчас?» →
{"query": "вид на жительство ВНЖ программа условия получения инвестиции сроки", "countries": [], "topic": "сроки и процесс", "survey": true, "intent": "knowledge"}
«как приготовить борщ?» →
{"query": "как приготовить борщ", "countries": [], "topic": "другое", "survey": false, "intent": "knowledge"}
«привет» →
{"query": "привет", "countries": [], "topic": "другое", "survey": false, "intent": "smalltalk"}
«спасибо большое!» →
{"query": "спасибо большое", "countries": [], "topic": "другое", "survey": false, "intent": "smalltalk"}
«что ты умеешь?» →
{"query": "что ты умеешь", "countries": [], "topic": "другое", "survey": false, "intent": "meta"}
«какие страны ты знаешь?» →
{"query": "какие страны ты знаешь", "countries": [], "topic": "другое", "survey": false, "intent": "meta"}
«привет! какой порог инвестиций на Мальте?» →
{"query": "минимальный порог инвестиций по программе Мальты", "countries": ["Мальта"], "topic": "инвестиции и стоимость", "survey": false, "intent": "knowledge"}
(вопрос по делу перевешивает приветствие)

Ответь ТОЛЬКО одним JSON без пояснений:
{"query": "полный вопрос", "countries": ["..."], "topic": "...", "survey": true|false, "intent": "knowledge"|"meta"|"smalltalk"}
